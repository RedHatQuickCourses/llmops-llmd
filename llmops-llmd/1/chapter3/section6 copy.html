llm-d Deployment Guide

Your Starting Point
This page provides a series of interactive (Arcade) walkthroughs to help you provision and configure your OpenShift environment with components to deploy the llm-d QuickStart using the Red Hat Demo Platform (RHDP). Follow these four steps in order, and you’ll be ready to deploy.

1. Lab Setup: Overview and Steps

This Arcade experience covers requesting or ordering the Lab Environment item from The Red Hat Demp Platform (RHDP) catalog.

2. Openshift Container Platform llm-d Checklist

This Arcade reviews the OCP Console steps or checklist to ensure an LLM-D compatible environment. It with logging into the OpenShift Container Platform Console and validating:

  • The OCP Version - needs to be 4.19+, which includes new Intelligent Inference Scheduler (API Gateway).

  • Validate the the GPU machineSet needs to be created, and ensure Worker nodes are ready.

  • Validate which Operators are installed in the cluster.

3. Getting Started in the OpenShift Cluster

This experience starts with a running RHDP environment and shows the steps to log into the environment and validate the MachineSets and worker nodes available to the cluster.

4. Configuring the Cluster (Operators)

This experience uses the AI BU Services GitHub repository to clone setup files. It shows how to:

  • Deploy the prerequisite installer for OpenShift 4.19

  • Manually install two Operators that where missing.
